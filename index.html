Architecture Claude - 5 couches

Chaque couche ajoute une capacite. Clique pour en savoir plus.

1
CLAUDE.md - Memory Layer
Le cerveau de ton agent. Lu en premier a chaque session.
  • Regles d'architecture et conventions
  • Catalogue de ressources (fichiers, templates, liens)
  • Instructions de comportement globales
  • Contexte entreprise et equipe
2
SKILLS - Knowledge Layer
Tes taches recurrentes automatisees en recettes reutilisables.
  • Description matching - invocation automatique
  • Contexte specifique a la tache
  • Exemples reels comme reference
3
HOOKS - Guardrail Layer
Garde-fous automatiques avant et apres chaque action.
  • PreToolUse - validation avant action
  • PostToolUse - verification apres action
  • Deterministe, pas de l'IA - comme des git hooks
4
SUBAGENTS - Delegation Layer
Instances specialisees pour deleguer des sous-taches.
  • Fenetre de contexte isolee par subagent
  • Roles : analyste, redacteur, verificateur...
  • Pas de recursion - un subagent ne spawne pas d'autres subagents
5
PLUGINS - Distribution Layer
Packages partageables : skills, agents, hooks, commands.
  • Organisation : skills/, agents/, hooks/, commands/
  • Partage entre equipes et projets
  • Marketplace de plugins communautaires

Premiers pas et choix du modele
30 min
Objectif

Maitriser l'interface Claude et comprendre quand utiliser Sonnet, Opus ou Extended Thinking.

A faire (pas a pas)
  • Va sur claude.ai et connecte-toi avec ton email Synapse
  • Cree un nouveau chat. Observe l'interface : historique a gauche, zone de chat au centre
  • Clique sur le selecteur de modele. 3 options : Sonnet (rapide), Opus (profond), Extended Thinking (raisonnement visible)
  • Tape la question ci-dessous avec chaque modele et compare les reponses
Prompt a copier
Analyse les avantages et inconvenients du teletravail total pour une entreprise tech de 90 personnes en France, sous convention Syntec.
Resultat attendu
Sonnet repond en ~15 sec avec une liste structuree.
Opus repond en ~45 sec avec plus de nuances et du contexte juridique Syntec.
Extended Thinking montre son raisonnement etape par etape avant de repondre.
Exemple pour ton pole
Tech : "Explique-moi les trade-offs entre REST et gRPC pour notre API medicamenteuse"
Revenue : "Resume les points de negociation cles de cette proposition de partenariat hospitalier"
Qara : "Compare les exigences ISO 13485 et ISO 27001 pour un SaaS medical"
Product : "Analyse cette user story et identifie les edge cases manquants"
Medical : "Synthetise les recommandations HAS sur les interactions medicamenteuses"
G&A : "Compare les implications CDD vs interim pour un remplacement de 6 mois sous Syntec"
Astuce - Par defaut, utilise Sonnet pour les questions rapides. Passe a Opus pour la profondeur (analyse juridique, strategie). Extended Thinking quand tu veux comprendre le raisonnement.
Exercice

Pose une question liee a ton activite reelle. Teste les trois modeles avec la meme question. Note les differences.

3
Artifacts et Projects
45 min
Objectif

Comprendre les Artifacts (documents vivants modifiables en conversation) et les Projects (contexte persistant entre sessions) dans Claude Chat.

C'est quoi un Artifact ? - Un artifact, c'est un document que Claude cree dans un panneau a droite de la conversation : tableau, texte structure, code, diagramme... Tu peux le modifier en discutant ("ajoute une colonne", "change le titre"), le telecharger, ou le copier. C'est un livrable vivant, pas un simple message.
A faire (pas a pas)
  • Demande a Claude de creer un artifact : "Cree un tableau de suivi projet avec objectifs, avancement, risques"
  • Modifie l'artifact en conversation : "Ajoute une colonne Impact" - observe la mise a jour en direct
  • Cree un Project : clique sur "Projects" dans la sidebar gauche, donne un nom
  • Ajoute 2-3 documents de contexte au Project (glisse-depose tes fichiers)
  • Pose une question dans le Project et observe comment Claude utilise le contexte
Projects Chat vs Projects Cowork - Attention, il y a deux types de Projects. Les Projects Chat (claude.ai) permettent d'uploader des documents et de garder du contexte entre les conversations. Les Projects Cowork (app desktop, qu'on verra au Niveau 2) pointent vers un dossier de ton disque et permettent a Claude de lire/ecrire des fichiers. Ce sont deux choses differentes. Les connecteurs configures dans Chat (Google Drive, etc.) ne sont pas automatiquement lies a Cowork.
Prompt a copier
Cree un tableau de bord projet avec : objectifs du trimestre, avancement par chantier (%), risques identifies, actions a prendre. Format : tableau structure avec code couleur.
Resultat attendu
Un artifact interactif avec un vrai tableau, modifiable en conversation. Tu peux ajouter des colonnes, changer des valeurs, reformater - tout en discutant.
Exemple pour ton pole
Project "Refacto API" avec la spec technique et le schema de base de donnees
Project "Compte CHU Bordeaux" avec l'historique des echanges et la proposition
Project "Audit ISO 13485" avec les documents de conformite et le plan d'audit
Project "Roadmap Q3" avec le backlog et les feedbacks utilisateurs
Project "Revue interactions" avec les publications et les donnees Theriaque
Project "Recrutement Data Engineer" avec la fiche de poste et la grille d'entretien
Astuce - Un Project bien alimente, c'est deja 50% du chemin vers un Claude personnalise. C'est le precurseur de l'arborescence Cowork qu'on verra au Niveau 2.
Exercice

Cree un Project pour un de tes chantiers en cours. Ajoute 2-3 documents. Pose une question et observe comment Claude utilise le contexte.

Prompt a copier
Lis tous les fichiers dans ce dossier. Pour chaque fichier, donne-moi : nom, type, et un resume en 1 phrase de ce qu'il contient.

Claude lit tes fichiers reels et produit un inventaire structure. Il supporte .docx, .xlsx, .pdf, .pptx, .csv et bien d'autres.

Construire ton espace Claude-Work
60 min
Objectif

Creer l'arborescence qui transforme Claude d'un assistant generique en TON assistant. Remplir les fichiers de contexte pour qu'il te connaisse. Tester que tout fonctionne.

C'est quoi CLAUDE.md ? - C'est un fichier texte place a la racine de ton dossier de travail. Claude le lit automatiquement en premier a chaque nouvelle session. Il contient ton identite, tes regles, ton contexte. C'est l'equivalent de dire a un nouvel assistant "voila qui je suis et comment je travaille" - sauf que tu ne le dis qu'une fois.
Prerequis - Prepare ces elements
  • Ton profil LinkedIn (URL ou copier-coller)
  • 3 a 5 messages (Slack, email) dont tu es satisfait du ton
  • Tes meilleurs templates ou livrables types
  • Le lien Notion ou wiki de ton equipe
Phase 1 - Creer l'arborescence
Prompt a copier
Cree l'arborescence suivante dans mon dossier Documents : Documents/Claude-Work/ .claude/ CLAUDE.md <- Le cerveau. Lu en premier a chaque session. skills/ <- Tes taches recurrentes automatisees (une skill = un SKILL.md + exemples) ABOUT-ME/ about-me.md <- Ton poste, parcours, competences, outils my-rules.md <- Ton ton, tes patterns interdits, tes formats. LE fichier le plus important. my-company.md <- Ta boite, ton equipe, tes outils internes OUTPUTS/ <- Tous les livrables produits par Claude, ranges par projet TEMPLATES/ <- Tes modeles de reference (presentations, CR, emails types) Puis demande-moi si j'ai des templates a copier dans TEMPLATES/ des maintenant.
Phase 2 - Remplir about-me.md
Prompt a copier
Voici mon profil LinkedIn [colle le lien ou le texte] et ma fiche de poste [colle ou donne le fichier]. Extrais-en un maximum d'infos : identite, parcours, poste, equipe, responsabilites, competences, outils. Si certains themes ne sont pas couverts (manager, localisation, domaines ou Claude peut m'aider le plus), utilise AskUserQuestion pour completer. Redige ensuite about-me.md toi-meme. Ne me donne pas un template vide. Montre-moi le resultat et demande "C'est fidele ? Tu veux ajuster quelque chose ?"
Phase 3 - Remplir my-rules.md (LE PLUS IMPORTANT)
Prompt a copier
Voici des messages que j'ai ecrits et dont je suis satisfait du ton [colle 3-5 messages Slack, mails ou Notion]. Analyse mon style en profondeur : longueur de phrase, ponctuation, emojis ou pas, tutoiement/vouvoiement, ouvertures, clotures, tics de langage, mots recurrents, niveau de familiarite. Si tu as besoin d'affiner, utilise AskUserQuestion. Puis construis my-rules.md. Sois synthetique mais AGRESSIF sur les patterns interdits. Sections : 1. Regles generales (langue, autonomie, validation, concision) 2. Mon ton et style - interne vs externe, avec des exemples reels tires de mes messages 3. Mots et expressions INTERDITS - liste precise 4. Patterns de reformulation INTERDITS - au moins 6 : pas de reformulation de ma question, pas de "C'est une excellente question", pas de recap non demande, pas de transitions pompeuses, pas de "N'hesite pas si tu as d'autres questions", pas de disclaimer non demande 5. Comportements INTERDITS - pas d'emojis non demandes, pas de sections supplementaires, pas d'allongement pour "faire complet" 6. Formats par livrable (Slack = court, Email = objet clair + CTA, Notion = headers) 7. Donnees et confidentialite (ne jamais inventer, citer ses sources) Mieux vaut trop de contraintes que pas assez. Montre-moi le resultat et demande "C'est assez strict ?"
Phase 4 - Remplir my-company.md
Prompt a copier
Je travaille chez Synapse Medicine. Va chercher les infos sur synapse-medicine.com et sur ce que tu connais deja de l'entreprise : secteur, activite, produits, taille, localisation, culture. Utilise AskUserQuestion pour completer ce que tu ne trouves pas : mon equipe (nom, taille, rattachement, mission), la convention collective applicable, et les outils internes (Slack, Notion, Google Workspace, Jira...). Redige my-company.md et montre-moi le resultat. Confirme les infos trouvees avant de les integrer.
Phase 5 - Completer CLAUDE.md
Prompt a copier
Lis les 3 fichiers ABOUT-ME/ et complete le CLAUDE.md pour en faire un resume executif de 40-60 lignes max. C'est le fichier que Claude lit en premier a chaque session - il doit etre court et pointer vers les fichiers de detail. Structure : 1. Identite (qui je suis, poste, equipe - 2-3 lignes) 2. Fichiers de reference - tableau : quel fichier lire selon le contexte 3. Regles cles - les 5-7 regles les plus importantes de my-rules.md 4. Mon ton (resume en 3-4 lignes) 5. Mots et patterns INTERDITS (les 5-10 plus critiques) 6. Outils (liste courte, une ligne par outil) Montre-moi le resultat.

Creer ton premier Skill
60 min
Objectif

Automatiser une tache recurrente en creant un skill base sur tes vrais livrables. Un skill, c'est un mini-mode d'emploi que Claude lit quand tu lui demandes un type de tache precis (CR de reunion, mail de refus candidat, brief produit, reporting...). La cle : partir de sources reelles, pas de regles abstraites. Plus les sources sont riches, plus la skill est fidele.

A faire
  • Phase 1 - Identifier : choisis une tache que tu fais au moins 2x par mois et formule la demande naturellement (ex : "fais-moi un CR", "prepare un mail de refus")
  • Phase 2 - Collecter : rassemble 2-3 exemples REELS de livrables finis (pas des templates vides). C'est LA phase qui fait la difference entre une skill generique et une skill excellente
  • Phase 3 - Construire : demande a Claude d'analyser tes exemples (structure, ton, patterns recurrents) et de creer le SKILL.md avec exemples de reference integres
  • Phase 4 - Tester : teste la skill sur un cas reel. Le premier jet n'est presque jamais parfait - c'est normal
  • Phase 4 bis - Iterer : ajuste le SKILL.md, re-teste, redemande du feedback jusqu'a satisfaction
Prompt Phase 1 - Identifier la tache
Je veux creer ma premiere skill. Aide-moi a identifier la bonne tache. Pose-moi des questions sur mes taches recurrentes (frequence, format, variantes) et propose-moi le meilleur candidat pour une skill.
Prompt Phase 2+3 - Collecter et construire
Voici [2-3] exemples reels de [type de livrable] que j'ai produits [colle-les ou donne les fichiers]. Analyse en profondeur : structure, ton, patterns recurrents, longueur, elements obligatoires, regles implicites. Puis cree une skill dans .claude/skills/[nom]/SKILL.md qui reproduit exactement mon style. Le SKILL.md doit contenir : principe fondamental, declencheurs typiques, structure du livrable, regles specifiques, anti-patterns a eviter, et un exemple de reference complet.
Prompt Phase 4 - Tester
Teste la skill que tu viens de creer sur ce cas concret : [decris le cas]. Produis le livrable en suivant le SKILL.md. Je vais comparer avec ce que j'aurais fait moi-meme.
Resultat attendu
Un dossier .claude/skills/[nom-de-ta-skill]/ avec un SKILL.md complet (principe, declencheurs, structure, regles, anti-patterns, exemple) + tes exemples sources dans un sous-dossier exemples/. Quand tu formules ta demande naturellement, Claude produit dans TON format et range dans OUTPUTS/.
Exemple pour ton pole
Skill "RFC technique" ou "Code review summary" base sur tes vraies RFCs
Skill "Note de synthese client" ou "Proposition commerciale" base sur tes vrais livrables
Skill "Rapport d'audit" ou "Non-conformite" base sur tes vrais rapports
Skill "Spec fonctionnelle" ou "Go/No-go analysis" base sur tes vraies specs
Skill "Revue de litterature" ou "Avis pharmaceutique" base sur tes vrais avis
Skill "Note de cadrage" ou "Reporting mensuel" base sur tes vrais reportings
Astuce - La qualite d'une skill depend a 80% de la qualite des sources. Plus tu fournis d'exemples reels, plus Claude capte les regles implicites (ce que tu fais systematiquement sans le verbaliser). Un bon SKILL.md contient toujours un "Principe fondamental" (l'approche, pas juste la tache) et des "Anti-patterns" (ce qu'il ne faut JAMAIS faire).
Structure du SKILL.md - Un bon SKILL.md suit cette structure : (1) Principe fondamental - 1-2 phrases sur l'approche (2) Declencheurs typiques - les formulations naturelles (3) Sources de reference - pointer vers le dossier exemples/ (4) Structure du livrable - sections detaillees (5) Regles specifiques - extraites de l'analyse (6) Anti-patterns - ce qu'il ne faut jamais faire (7) Exemple de reference complet.
Exercice

Identifie ta tache la plus recurrente. Rassemble 2-3 livrables finis. Utilise les prompts ci-dessus dans l'ordre (Phase 1, puis 2+3, puis 4). Itere jusqu'a ce que le resultat corresponde a ce que tu aurais fait toi-meme.

10
Hooks - Garde-fous automatiques
40 min
Objectif

Mettre en place des regles automatiques qui s'executent avant ou apres les actions de Claude. Ce n'est pas de l'IA, c'est du code deterministe - comme des git hooks.

C'est quoi un hook ? - Un hook, c'est un script qui se declenche automatiquement quand un evenement se produit. Par exemple : "quand Claude attend mon input, envoie-moi une notification macOS". Ca se configure dans le fichier .claude/settings.json - un fichier de configuration technique qu'on n'avait pas encore touche.
A faire
  • Comprends le principe : Evenement > Filtre > Action automatique
  • Demande a Claude de creer ton premier hook avec le prompt ci-dessous
  • Verifie que le fichier .claude/settings.json a ete cree
  • Teste en lancant une tache et en attendant que Claude te notifie
Prompt a copier
Cree un hook simple dans .claude/settings.json qui m'envoie une notification macOS quand tu attends mon input. Explique-moi ce que tu as fait.
Resultat attendu
Un fichier .claude/settings.json avec un hook Notification fonctionnel. Tu recois une notification macOS chaque fois que Claude attend ton input.
Astuce - Les hooks RENFORCENT les restrictions. Une instruction dans CLAUDE.md est une demande. Un hook est une application. Si une regle doit tenir a chaque fois, fais-en un hook. Pour les demandes complexes, tu peux aussi creer un hook /plan qui active le mode planification de Claude.
Exercice

12
Agent Teams
60 min
Objectif

Orchestrer des equipes d'agents qui travaillent en parallele avec communication bidirectionnelle.

Subagent vs Agent Team - Un subagent rapporte au principal (unidirectionnel). Un agent team = plusieurs agents qui communiquent entre eux (bidirectionnel). C'est la difference entre deleguer une tache et orchestrer une equipe.
A faire
  • Concois un workflow multi-agent sur un cas concret de ton metier
  • Lance 2-3 agents en parallele avec des roles clairs
  • Orchestre la fusion des resultats
  • Mesure le gain de temps vs execution sequentielle
Prompt a copier
Je dois preparer une presentation complete sur [sujet]. Lance une equipe de 3 agents : Agent 1 = recherche et donnees. Agent 2 = redaction et structure. Agent 3 = mise en forme et visuels. Coordonne-les et fusionne le resultat.
Exemple pour ton pole
Agent 1 ecrit les tests, Agent 2 refactore, Agent 3 met a jour la doc
Veille concurrentielle > analyse > deck comparatif > recommandations
Revue documentaire > gap analysis > plan de remediation > rapport
Research users > specs > maquettes > user stories
Revue litterature > extraction donnees > synthese > recommandations
Collecte donnees > reporting > analyse > presentation staff meeting
Astuce - Le secret n'est pas le nombre d'agents mais la qualite du decoupage. Des taches reellement independantes = gain de temps. Des taches interdependantes = overhead de coordination.
Exercice

Choisis un livrable qui te prend 2-3 heures. Decoupe en 3 sous-taches. Lance un agent par sous-tache.
